Git Merge Conflicts

When you merge branches in Git, you may run into a merge conflict. This happens when you merge branches that have competing changes, and Git needs your help to decide which changes to incorporate in the final merge.

When a merge conflict occurs, Git will tell you that a file has a conflict, and it will show you the conflicting changes. For example:

<<<<<<< HEAD
This is the original text.
=======
This is the conflicting text that was added in the branch you're merging in.
>>>>>>> branch-name

To resolve the conflict, you need to edit the file to remove the conflict markers and conflicting lines, and then commit the changes. After that, you can complete the merge.

It's important to resolve merge conflicts carefully, as they can cause errors in your code if not handled correctly. It's a good idea to review the changes and test your code after resolving a merge conflict to make sure everything is working as expected.

Demonstration

Let's simulate a merge conflict by making conflicting changes to a file in two branches and then merging the branches together. We can practice this using the merge_conflict_test.txt file.

Here are the steps to follow using the command line:

  1. Create a new branch called change_1 
    git checkout -b change_1
  2. Change the second line of the merge_conflict_test.txt to some other text. You can copy the latest observation for KLWT from Aviation Weather Center and paste that over the current observation in the file.
  3. Commit the changes to the change_1 branch and push the branch to the remote repository. 
    git add . ; git commit -m "Updated observation in change_1 branch" ; git push origin change_1
  4. Switch back to the main branch and create a new branch called change_2 
    git checkout main; git checkout -b change_2
  5. Change the second line of the merge_conflict_test.txt to some other text than what you did in the change_1 branch. You can copy the SECOND observation for KLWT from Aviation Weather Center and paste that over the current observation in the file.
  6. Commit the changes to the change_2 branch and push the branch to the remote repository. 
    git add . ; git commit -m "Updated observation in change_2 branch" ; git push origin change_2
  7. Switch back to the main branch and merge the change_1 branch into the main branch. 
    git checkout main; git merge change_1
  8. Switch back to the main branch and merge the change_2 branch into the main branch. 
    git checkout main; git merge change_2
  9. Resolve the merge conflict in the merge_conflict_test.txt file by editing the file to remove the conflict markers and conflicting lines. You can choose to keep one of the observations or combine them as you see fit.
  10. Commit the changes to resolve the merge conflict. 
    git add . ; git commit -m "Resolved merge conflict" ; git push origin main
